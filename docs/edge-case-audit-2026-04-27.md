# Edge-case audit — 2026-04-27

Read-only systematic walk over the 5 primitives + the unified CLI + shared
infra + bundled hooks. Each row identifies one untested or unhandled edge
case with a concrete reproduction and a suggested fix.

Out of scope (deliberately):

- Behaviour that is documented as an intentional limitation (e.g.
  `swanlake bench --quick` requires the `bench/` dir; the public repo
  does not ship it).
- The reconciler subpackage — covered by its own test suite under
  `reconciler/tests/`.
- Anything that requires the operator's local DEFENSE-BEACON checkout
  to reproduce.

Severity scale:

- **blocker** — security or data-loss risk in a default code path.
- **high** — correctness defect that bites a non-pathological operator
  workflow.
- **medium** — surprising behaviour, drift between modules, or a
  fail-open path that should fail-closed.
- **low** — small bug with a 1-5 line fix and an obvious test.
- **nit** — cosmetic, hygiene, or dead code.

This audit ships the **low** and **nit** fixes in the same PR. The
**medium** and above need design discussion and are deferred to follow-up
PRs.

## Notes on what was checked but found OK

- `_optout.find_marker` walk depth is bounded to 32; symlink-loop safe.
- `state.ensure_state_root` chmod is wrapped in try/except for FUSE filesystems.
- `audit._atomic_append` uses fcntl.flock — concurrent appends are serialised.
